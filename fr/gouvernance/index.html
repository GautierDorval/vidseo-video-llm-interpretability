Gouvernance

Versionnement, contrôle des changements, politique de revue et règles de cohérence du dépôt de référence VidSEO.

Le but de la gouvernance est simple : maintenir stable le sens de VidSEO à travers le temps et les surfaces.

Classes de contenu

Ce dépôt utilise trois classes de contenu :

  • normatif : définit le sens, la portée, les exclusions ou des règles requises ;
  • informatif : explique ou contextualise sans redéfinir le sens ;
  • contextuel : pointe vers des ressources externes ou du matériel adjacent.

Une page ne doit jamais glisser silencieusement d’une intention informative vers une intention normative.

Identifiants stables

Une fois publiés, ces identifiants doivent rester stables :

  • les identifiants de documents (VIDSEO-DOC-*) ;
  • les identifiants de termes (VIDSEO-TERM-*) ;
  • les codes de termes (VIDSEO-T-*) ;
  • les slugs canoniques lorsqu’ils peuvent déjà être liés depuis l’extérieur.

Classes de changements

Patch

Corrections de coquilles, clarification rédactionnelle, nettoyage de métadonnées ou amélioration de mise en page sans changement de sens.

Mineur

Nouveau matériel informatif, nouvelles pages de termes, ajouts de traduction ou surfaces machine enrichies sans modification du périmètre fondamental.

Majeur

Tout changement touchant :

  • la définition du produit ;
  • la hiérarchie d’autorité ;
  • les exclusions ;
  • la frontière interprétative ;
  • la stratégie d’URL canoniques.

Les changements majeurs exigent une augmentation de version, une entrée dans le journal des changements et une revue complète de cohérence.

Exigences de contrôle de dérive

Si une modification affecte le sens du produit ou la découverte canonique, les contributeurs doivent mettre à jour, selon le cas :

  • VERSION
  • CHANGELOG.md
  • meta.json
  • ai-manifest.json
  • links.json
  • data/documents.json
  • data/terms.json
  • data/capabilities.json
  • llms.txt
  • sitemap.xml

Règle de revue

Aucune contribution ne doit être fusionnée si elle introduit :

  • des promesses de classement ;
  • des promesses de citation ;
  • des revendications cachées de capacités IA ;
  • des changements d’autorité non expliqués ;
  • des surfaces machine cassées ;
  • une navigation canonique non traduite lorsque la parité bilingue est requise.

Modèle d’autorité

L’autorité finale sur le sens normatif reste entre les mains du mainteneur des surfaces canoniques.

Surfaces canoniques

  • GitHub repository
  • WordPress plugin page

Surfaces machine
