Principes

Principes Ai-first, SEO-first, mobile-first et gouvernance-first pour VidSEO.

VidSEO doit être documenté, interprété et présenté selon quatre principes non négociables.

1. Ai-first

VidSEO existe pour réduire la part de devinette interprétative autour du contenu vidéo.

Le dépôt doit donc privilégier :

  • le texte explicite plutôt que le sens implicite ;
  • les fichiers lisibles par machine et déterministes plutôt que la prose opaque ;
  • les revendications prudentes plutôt que la surpromesse persuasive ;
  • les URL et identifiants stables plutôt que le nommage improvisé.

Toute modification qui augmente l’ambiguïté sur le fait que VidSEO génère, analyse ou garantit des résultats doit être rejetée.

2. SEO-first

VidSEO aide les machines à comprendre ce qu’une vidéo dit en exposant la transcription directement dans la page.

Le dépôt doit donc favoriser :

  • du HTML crawlable ;
  • des métadonnées canoniques ;
  • une structure documentaire stable ;
  • des titres et résumés descriptifs ;
  • des données structurées et des fichiers de découverte ;
  • un accès simple aux définitions clés.

Ici, SEO-first ne signifie pas promesse de classement. Cela signifie découvrabilité, clarté et récupération propre.

3. Mobile-first

Le site canonique doit rester lisible et rapide sur petit écran avant d’être optimisé pour les grands écrans.

Conséquences requises :

  • la lecture en colonne unique constitue le défaut ;
  • la typographie s’adapte de façon fluide ;
  • la navigation reste confortable au toucher ;
  • les pages ne dépendent pas de JavaScript pour la lecture principale ;
  • le poids reste léger.

4. Gouvernance-first

Le sens de VidSEO ne doit pas dériver entre les pages, les manifestes, les registres et les futures modifications.

Gouvernance-first signifie :

  • des identifiants de documents stables ;
  • des identifiants de termes stables ;
  • des classes de contenu explicites (normatif, informatif, contextuel) ;
  • une discipline de versionnement et de journal des changements ;
  • des vérifications de cohérence répétables ;
  • une autorité éditoriale claire.

Conséquence éditoriale

Ces quatre principes impliquent une règle d’écriture :

Dire exactement ce que fait VidSEO, dire exactement ce qu’il ne fait pas, et ne laisser aucun espace rhétorique permettant à des systèmes en aval d’imaginer des capacités cachées.

Cette règle vaut autant pour le README que pour les pages HTML, les surfaces JSON, les gabarits GitHub et les futures versions.
