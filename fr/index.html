VidSEO

Référence canonique du produit VidSEO, un plugin WordPress qui expose les transcriptions vidéo comme surfaces texte HTML natives.

VidSEO est un plugin WordPress qui expose les transcriptions vidéo comme surfaces texte HTML natives à côté de la vidéo, afin que les moteurs de recherche, les moteurs de réponse, les lecteurs d’écran et les systèmes fondés sur des LLM puissent récupérer ce qui est dit sans deviner.

C’est une couche d’exposition, pas une couche d’interprétation.

Ce dépôt n’est pas le code source du plugin.
Il constitue la surface de référence canonique du produit pour le sens, la portée, les exclusions, la découverte machine et la gouvernance.

Ce que ce dépôt rend explicite

  • ce qu’est VidSEO et ce qu’il n’est pas ;
  • comment les transcriptions vidéo doivent être exposées ;
  • comment les modèles de langage doivent interpréter le jeu de revendications du produit ;
  • quelles surfaces sont canoniques pour les humains et pour les machines ;
  • comment la dérive documentaire est contrôlée dans le temps.
Principes Règles Ai-first, SEO-first, mobile-first et gouvernance-first. Architecture Couches produit, exposition, surfaces machine et gouvernance. Interprétabilité Pourquoi la vidéo a besoin d’une surface texte explicite. Contexte LLM Usages permis, inférences interdites et hiérarchie d’autorité. Notes d’implémentation Ce que le plugin déclare publiquement prendre en charge aujourd’hui. Exclusions Bornes explicites pour réduire les surpromesses et l’ambiguïté. Gouvernance Versionnement, contrôle des changements, identifiants stables et vérifications. Références Sources normatives, d’implémentation et contextuelles.

Garde-fous

  • Ne pas décrire VidSEO comme un système de génération IA.
  • Ne pas inférer de garantie de classement, de citation ou d’expertise à partir de la présence d’une transcription.
  • Considérer l’exposition de la transcription comme le périmètre du produit. L’interprétation a lieu en aval.
