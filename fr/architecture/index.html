Architecture

Architecture conceptuelle de VidSEO : couches produit, exposition, surfaces machine et gouvernance.

Deux architectures importent pour VidSEO :

  1. l’architecture produit qui expose les transcriptions dans une page WordPress ;
  2. l’architecture de référence qui définit la manière dont le produit est décrit et découvert.

Couche 1. Couche produit

Les capacités décrites publiquement indiquent que VidSEO peut :

  • intégrer des sources vidéo prises en charge ;
  • associer une transcription à une vidéo donnée ;
  • rendre cette transcription en HTML dans la page ;
  • maintenir la transcription proche du contexte de la vidéo.

Il s’agit de la couche produit opérationnelle.

Couche 2. Couche d’exposition

La revendication critique n’est pas une « magie SEO vidéo ».

C’est l’exposition textuelle.

La transcription devient une surface de premier niveau dans le DOM au lieu de rester un contenu audiovisuel opaque. C’est cette couche qui compte pour les crawlers, les moteurs de réponse et les systèmes fondés sur la récupération.

Couche 3. Couche des surfaces machine

Ce dépôt ajoute un second système d’exposition autour du produit lui-même :

  • meta.json
  • ai-manifest.json
  • links.json
  • llms.txt
  • data/documents.json
  • data/terms.json
  • data/capabilities.json

Ensemble, ces fichiers réduisent l’ambiguïté sur le sens, la portée et l’autorité du produit.

Couche 4. Couche de gouvernance

Un sens sans gouvernance dérive.
Une gouvernance sans surfaces machine reste implicite.

La couche de gouvernance comprend donc :

  • le suivi de version dans VERSION ;
  • l’historique des versions dans CHANGELOG.md ;
  • la terminologie stable dans TERMS.md ;
  • l’autorité éditoriale dans CODEOWNERS ;
  • les règles de contribution dans CONTRIBUTING.md ;
  • les vérifications de cohérence dans scripts/verify_consistency.py ;
  • l’application en CI dans .github/workflows/consistency.yml.

Pourquoi cette architecture compte

Un plugin peut être utile tout en restant mal interprété par les machines.
Un dépôt peut être élégant tout en restant sémantiquement instable.

Cette architecture cherche à éviter ces deux échecs :

  • le produit expose le sens de la vidéo sous forme de texte ;
  • le dépôt expose le sens du produit sous forme de données de référence gouvernées.
