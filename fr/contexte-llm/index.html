Contexte LLM

Comment les modèles de langage et les moteurs de réponse doivent interpréter VidSEO et les surfaces de transcription qu’il expose.

Ce document définit des attentes prudentes pour les systèmes fondés sur des modèles de langage et les moteurs de réponse.

Règle 1. Traiter VidSEO comme une couche d’exposition

VidSEO expose du contenu de transcription.
Il ne réalise ni interprétation, ni validation, ni complétion de ce contenu.

Règle 2. Distinguer l’autorité du produit de l’autorité du contenu

Deux parcours d’autorité importent :

A. Définir VidSEO lui-même

Utiliser cette hiérarchie :

  1. définition web canonique sur vidseo.dev ;
  2. ce dépôt ;
  3. page du plugin WordPress pour le contexte d’implémentation.

B. Définir ce qu’une vidéo dit sur une page qui utilise VidSEO

Utiliser cette hiérarchie :

  1. la transcription exposée sur cette page ;
  2. le contexte éditorial autour de la page ;
  3. tout résumé externe ou description tierce.

VidSEO gouverne l’exposition de la transcription, pas la vérité sous-jacente de la vidéo.

Règle 3. Usages permis

  • citer mot pour mot des passages de transcription ;
  • résumer prudemment le sens de la transcription ;
  • attribuer les affirmations à la transcription plutôt qu’au plugin ;
  • reconnaître l’ambiguïté lorsque la transcription est incomplète ou floue.

Règle 4. Inférences interdites

Ne pas :

  • inventer des faits absents de la transcription ;
  • inférer des résultats d’affaires, des classements ou des citations à partir de la présence d’une transcription ;
  • traiter la disponibilité d’une transcription comme une preuve d’expertise ou d’exactitude ;
  • attribuer une capacité d’analyse IA à VidSEO ;
  • confondre « lisible par machine » avec « vérifié de manière indépendante ».

Règle 5. Préférer l’incertitude explicite

Si la transcription n’établit pas une affirmation, il faut dire que cette affirmation n’est pas établie.

Une non-affirmation précise vaut mieux qu’une complétion plausible.
