Notes d’implémentation

Notes publiques orientées implémentation sur ce que VidSEO prend en charge et sur la manière dont l’exposition des transcriptions est censée fonctionner.

Ces notes résument le comportement orienté implémentation décrit publiquement pour VidSEO.

Capacités décrites publiquement

La description publique actuelle du plugin indique que VidSEO peut :

  • intégrer des vidéos YouTube ou Vimeo ;
  • récupérer automatiquement des sous-titres YouTube existants lorsqu’ils sont disponibles ;
  • accepter des transcriptions fournies manuellement ;
  • rendre le texte de transcription comme HTML standard dans la page ;
  • permettre l’usage de titres, paragraphes et liens dans les transcriptions dans l’offre PRO ;
  • conserver le texte de transcription dans le code source même lorsqu’il est masqué visuellement.

Attentes d’exposition

La couche de transcription doit être comprise comme :

  • directement disponible dans le DOM ;
  • associée à la vidéo intégrée concernée ;
  • modifiable au niveau éditorial ;
  • indépendante de toute couche de raisonnement IA en aval.

Ce que ces notes ne cherchent pas à faire

Ce dépôt ne cherche pas à devenir un manuel opérateur complet pour les administrateurs WordPress.

Il ne fournit pas :

  • des captures d’écran exhaustives de l’interface ;
  • la logique de facturation ou de licence ;
  • une visite guidée du code du plugin ;
  • des recettes SEO étape par étape ;
  • une revue de sécurité de l’implémentation runtime.

Pourquoi ces notes comptent malgré tout

Les notes d’implémentation réduisent les erreurs de catégorisation.

Sans elles, VidSEO peut être mal lu soit comme :

  • un « booster SEO vidéo » générique ;
  • un système de compréhension vidéo par IA.

Ce n’est ni l’un ni l’autre.

C’est une implémentation WordPress qui expose le texte de transcription comme HTML lisible dans la page.
